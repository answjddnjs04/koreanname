# K-Soul Name 프로젝트 개발 가이드 (MVP 단계)

**프로젝트명**: K-Soul Name (한국 이름 생성기)
**목표**: 이미지 생성 등 복잡한 기능 없이, 오직 12개의 질문에 대한 답변을 분석하여 사용자에게 가장 잘 어울리는 한국식 이름과 그 의미를 제공하는 MVP(Minimum Viable Product) 구축.
**배포 플랫폼**: Vercel 또는 Cloudflare Pages
**기술 스택**:
- Next.js 15 (App Router)
- Tailwind CSS (심플하고 깔끔한 UI)
- Zustand (퀴즈 상태 관리)
- Anthropic Claude API (답변 분석 및 이름 생성 로직)

---

## 🚀 MVP 개발 10단계 로드맵

### Step 1: 프로젝트 초기 설정 (30분)
1. Next.js 프로젝트 생성 (`npx create-next-app@latest`)
2. 필요한 라이브러리 설치 (`zustand`, `lucide-react`, `shadcn/ui`)
3. 기본 폴더 구조 정리 (`app/`, `components/`, `store/`, `lib/`)

### Step 2: 12가지 질문 설계 (1시간)
1. 사용자의 성향, 취향, 가치관을 파악할 수 있는 12개 질문 확정.
2. 각 질문당 4~5개의 선택지 구성.
3. 질문 데이터를 JSON 형태로 정리 (`lib/questions.ts`).

### Step 3: 상태 관리 시스템 구축 (30분)
1. Zustand를 사용하여 유저의 답변을 저장할 Store 생성.
2. 현재 질문 번호, 답변 배열, 결과 데이터 등을 관리.

### Step 4: 퀴즈 인터페이스(UI) 구현 (2시간)
1. 한 번에 하나의 질문만 보여주는 카드 타입 UI 설계.
2. 선택지 클릭 시 다음 질문으로 넘어가는 애니메이션/로직 구현.
3. 상단에 진행률 표시줄(Progress Bar) 추가.

### Step 5: 답변 수집 및 검증 로직 (1시간)
1. 12개 질문에 모두 답했는지 확인하는 유효성 검사.
2. 수집된 답변 데이터를 서버로 전송하기 위한 포맷팅.

### Step 6: 이름 생성 알고리즘/프롬프트 설계 (2시간)
1. Claude API에 전달할 시스템 프롬프트 작성.
2. "12개의 답변을 바탕으로 성씨, 이름, 한자 의미, 성격 분석"을 포함하도록 설정.
3. (선택사항) API 비용 절감을 위해 고정된 로직 기반의 매칭 시스템 고려.

### Step 7: API Route 생성 (1시간)
1. `app/api/generate-name/route.ts` 생성.
2. 프론트엔드에서 받은 답변을 Claude API에 전달하고 결과를 받아오는 로직 구현.

### Step 8: 결과 표시 페이지 구현 (2시간)
1. 생성된 이름(한글/한자/로마자)을 크고 아름답게 표시.
2. 이름에 담긴 의미와 사용자의 성향 분석 텍스트 출력.
3. "다시 하기" 버튼 추가.

### Step 9: 텍스트 및 카피라이팅 최적화 (1시간)
1. 질문과 결과의 문구들을 매끄럽고 흥미롭게 수정.
2. 한국어/영어 병기 여부 결정 및 적용.

### Step 10: 최종 테스트 및 배포 (1시간)
1. 모바일/데스크탑 반응형 UI 최종 점검.
2. Vercel 또는 Cloudflare Pages를 통한 초고속 배포.

---

## ✅ 개발 원칙
- **이미지 없음**: MVP 단계에서는 텍스트와 레이아웃만으로 충분한 가치를 전달합니다.
- **속도 우선**: 복잡한 기능보다 퀴즈의 흐름이 끊기지 않는 것이 중요합니다.
- **정확한 매칭**: 질문의 의도가 결과 이름에 잘 반영되도록 프롬프트를 세밀하게 조정합니다.
